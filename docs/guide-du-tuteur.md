# Guide du tuteur

Ce guide aide le tuteur à animer le stage sans transformer les deux semaines en cours magistral.

## Posture recommandée

Le tuteur joue le rôle d'un tech lead bienveillant :

- il donne un cadre clair ;
- il explique les objectifs métier ;
- il laisse les élèves chercher ;
- il relit leur code ;
- il valorise les progrès ;
- il insiste sur la confidentialité et la responsabilité.

## Rythme quotidien conseillé

Chaque journée peut suivre ce rythme :

1. **Accueil et objectif du jour** — 15 minutes
2. **Démonstration courte** — 20 à 30 minutes
3. **Exercice guidé** — 1 heure
4. **Pause et échange métier** — 15 minutes
5. **Travail en autonomie** — 1 à 2 heures
6. **Revue de code ou restitution** — 30 minutes
7. **Bilan du jour** — 10 minutes

## Règles importantes

### Données

Ne jamais utiliser de données réelles de clients, d'entreprises ou de collaborateurs.

Toutes les données utilisées dans ce dépôt sont fictives.

### IA

Présenter l'IA comme un assistant :

- elle peut résumer ;
- elle peut reformuler ;
- elle peut proposer des pistes ;
- elle peut se tromper ;
- elle ne doit pas décider à la place d'un humain.

### Finance

Rester volontairement simple. Les élèves n'ont pas besoin de comprendre la comptabilité avancée.

Les notions utiles sont :

- entreprise ;
- chiffre d'affaires ;
- effectif ;
- demande de financement ;
- dossier complet ou incomplet ;
- risque apparent ;
- résumé pour analyste.

## GitHub comme support pédagogique

Utiliser GitHub pour montrer le fonctionnement d'une équipe :

- une issue correspond à une mission ;
- une branche correspond au travail d'un stagiaire ;
- une pull request permet la revue ;
- un test automatique vérifie que le code fonctionne.

## Ce qu'il faut éviter

- Donner trop de théorie au début.
- Corriger tout à la place des élèves.
- Utiliser des données réelles.
- Présenter l'IA comme magique.
- Chercher la perfection technique plutôt que la compréhension.

## Évaluation finale

Le dernier jour, les élèves peuvent répondre à ces questions :

1. Qu'avez-vous appris sur Python ?
2. Qu'avez-vous compris du métier de développeur ?
3. À quoi sert GitHub ?
4. Comment l'IA peut-elle aider un analyste ?
5. Quelles sont les limites ou les précautions à prendre ?
